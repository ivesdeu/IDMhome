WordPress theme upload (fix "missing style.css")
===============================================

WordPress requires style.css inside the THEME FOLDER at the root of the ZIP.

CORRECT package layout (what you must upload):
  ibmcustom/
    style.css          ← required (Theme Name header is in this file)
    functions.php
    index.php
    ... (other theme files)

WRONG (common mistakes):
  - Zipping the whole "wp-theme" folder → WordPress looks for wp-theme/style.css (missing).
  - Zipping only the files inside ibmcustom without the "ibmcustom" folder → broken layout on some hosts.
  - Using an old zip that was created before style.css existed.

Recommended: from the repo root run:
  npm run zip-theme

That creates: ibmcustom-wp-theme.zip in the project root. Upload THAT file in
WP Admin → Appearance → Themes → Add New → Upload Theme.

Verify anytime:
  npm run verify-theme-zip

Styles must match the React app (transfer / deploy)
====================================================

The theme loads compiled utilities from:
  ibmcustom/assets/css/tailwind-built.css

That file is built from class names found in:
  - repo src/ (React)
  - all *.php in the theme
  - theme assets/js/

Always rebuild before creating the zip:
  npm run build:wp-css
  npm run zip-theme

(zip-theme runs build:wp-css automatically.)

Use the SAME Tailwind version as the app: @tailwindcss/cli is pinned to match
tailwindcss / @tailwindcss/vite so WordPress CSS does not drift from local Vite.
